LaunchDarkly.Client Namespace

This is the main namespace for the LaunchDarkly .NET SDK. You will most often use LdClient (the SDK client), Configuration (configuration options for the client), and User (user properties for feature flag evaluation).
Classes
  ClassDescription
Components
Provides factories for the standard implementations of LaunchDarkly component interfaces.
Configuration
Configuration options for LdClient. This class should normally be constructed with Builder(String).
ConfigurationExtensions Obsolete.
Extension methods that can be called on a Configuration to add to its properties. It is preferable to use Builder(String) instead.
FeatureFlagsState
A snapshot of the state of all feature flags with regard to a specific user. See calling AllFlagsState(User, FlagsStateOption).
FeatureStoreCacheConfig
Parameters that can be used for IFeatureStore database integrations that support local caching.
FlagsStateOption
Optional parameters that can be passed to AllFlagsState(User, FlagsStateOption).
ILdClientExtensions
Convenience methods that extend the ILdClient interface.
InMemoryFeatureStore
In-memory, thread-safe implementation of IFeatureStore.
LdClient
A client for the LaunchDarkly API. Client instances are thread-safe. Applications should instantiate a single LdClient for the lifetime of their application.
VersionedDataKindT
Objects used by IFeatureStore implementations to denote a specific collection of IVersionedData-derived objects.
Interfaces

IConfigurationBuilder
A mutable object that uses the Builder pattern to specify properties for a Configuration object.
IEventProcessorFactory
Interface for a factory that creates some implementation of IEventProcessor.
IFeatureStore
Interface for a data store that holds feature flags and related data received by the streaming client.
IFeatureStoreFactory
Interface for a factory that creates some implementation of IFeatureStore.
ILdClient
Interface defining the public methods of LdClient.
IUpdateProcessor
Interface for an object that receives updates to feature flags, user segments, and anything else that might come from LaunchDarkly, and passes them to an IFeatureStore.
IUpdateProcessorFactory
Interface for a factory that creates some implementation of IUpdateProcessor.
IVersionedData
Common interface for string-keyed, versioned objects that can be kept in an IFeatureStore.
IVersionedDataKind
Objects used by IFeatureStore implementations to denote a specific collection of IVersionedData-derived objects.
IVersionedDataOrdering
This interface is implemented by IVersionedDataKind instances that specify a preferred ordering for data updates.
